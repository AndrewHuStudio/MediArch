### C. 真相核实：存在“vlm_processed=true 但实际上无描述”的历史数据（已定位、已修正工具口径）
你提到的“VLM 根本没有执行/描述全为空”的结论 **对《医院建筑设计指南》这类资料是成立的**，但需要澄清两点：
1) 我们库里并没有使用 `vlm_description` 字段；**图片语义描述实际存放在 chunk 的 `content` 字段里**。  
2) 历史数据里存在大量“`metadata.vlm_processed=true` 但 `content` 仍是占位符”的情况，导致 `audit_mongo_doc` 以前按 `vlm_processed` 统计会产生误导。  

#### 复核方法（可复现）
1) 用审计 CLI 直接复核（新版口径：`image_vlm_ok` 以 `content` 是否包含 `] + 描述` 为准，同时保留 `image_vlm_flag_true`）：
```powershell
python -m backend.cli.audit_mongo_doc --doc-id 6944bd320e5acfd26a8509cc --limit 1
```
关键输出（已核实）：
- `image_with_url=515`
- `image_vlm_flag_true=515`
- `image_vlm_ok=0`
- `image_content_len_gt_80=0`

2) 用 Python 直接验证“所有图片 chunk 的 content 都是同一个占位符长度”（不依赖 Mongo 查询语法）：
```powershell
python -c "from pymongo import MongoClient; from bson import ObjectId; import os; uri=os.getenv('MONGO_URI','mongodb://admin:mediarch2024@localhost:27017/'); db=MongoClient(uri)[os.getenv('MONGO_DB','mediarch')]; oid=ObjectId('6944bd320e5acfd26a8509cc'); cur=db.mediarch_chunks.find({'doc_id':oid,'content_type':'image'},{'content':1,'metadata.vlm_processed':1}); lens=[]; flag=0; n=0; \nfor d in cur:\n n+=1\n if (d.get('metadata') or {}).get('vlm_processed'):\n  flag+=1\n lens.append(len(d.get('content') or ''))\nprint('images',n); print('vlm_flag_true',flag); print('min_len',min(lens)); print('max_len',max(lens)); print('avg_len',sum(lens)/len(lens))"
```

#### 影响解释（为什么“搜护理单元搜不到图/出图不相关”）
- 由于图片 chunk 的 `content` 仍是占位符（例如 `[图片: [图片: 无标题]]`），图片向量语义几乎为 0。  
- 检索阶段更容易命中封面/扉页之类的“弱相关图片”，出现“正文引用页码正确，但图片不匹配”。  

#### 修正动作（已完成）
- `backend/cli/audit_mongo_doc.py`：新增区分 `image_vlm_flag_true` vs `image_vlm_ok`，避免误判。  
- `backend/cli/vlm_doc_status.py` / `backend/cli/vlm_manager.py`：统计口径改为基于 `content` 的“真实描述”判断；同时保留 `flag_true` 用于发现历史脏标记。  
- `backend/cli/vlm_backfill_images.py`：回填筛选不再依赖 `metadata.vlm_processed`，而改为判断 `content` 是否已有真实描述；因此 **不需要 `--force` 也能修复历史脏标记资料**。

## 🧭 设计判断（与你的思路对齐）

你提出的关键点成立：**终端用户不会知道后台资料叫什么**，因此系统不应要求用户写“给我返回《某某资料》”。更符合用户心智的路径是：
- **先让图片 chunk 具备足够强的语义（VLM caption）**，支持“全库语义找图”；  
- “资料范围限定（doc scoping）”作为**调试/高级筛选**能力保留，而不是用户必须输入资料名；  
- 通过“VLM 覆盖面 + 检索排序策略”从根本上缓解漂移与选图不稳。
