#### [OK] 深度分析现有架构，制定优化方案

**分析成果**：
通过 Explore Agent 对项目进行了全面架构分析，生成了详细的优化方案报告。

**核心发现**：

1. **检索效率问题**：
   - Neo4j 检索深度浅（depth=2），实体覆盖不足
   - Agent 间信息隔离，Neo4j 扩展的实体未被 Milvus/MongoDB 利用
   - 检索结果数量少（6条左右）

2. **双向连接完整性问题**：
   - Neo4j → Chunks: ✅ 完整（chunk_ids 属性）
   - Chunks → Neo4j: ⚠️ entity_ids 类型可能错误
   - 空间关系缺少 chunk 追踪

3. **Knowledge Fusion 缺失**：
   - 项目计划中有 fusion.py，但功能不完整
   - 缺少 unified_hints 生成逻辑
   - 缺少 answer_graph_data 可视化数据结构

**优化方案**（三阶段）：

**阶段 1：快速修复（1-2天）** 🥇
- 修复 entity_ids 类型问题
- 提升 Neo4j 检索深度（depth: 2→3, k_edges: 100→200）
- 补充空间关系的 chunk 追踪

**阶段 2：核心功能增强（1-2周）** 🥈
- 实现 Knowledge Fusion 模块
- 重构 Supervisor Graph 流程（真正的并行 + 融合）
- 优化 MongoDB Agent 检索

**阶段 3：精细化优化（1个月）** 🥉
- 添加 chunk ↔ attribute 连接
- 前端图谱可视化集成
- PDF 精确高亮功能
