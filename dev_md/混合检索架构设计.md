# 混合检索架构：向量 + 图谱互补方案

## 核心思想

**向量检索**和**图谱检索**同时进行，互为补充，形成完整的知识网络。

---

## 架构设计

### 1. 数据层（统一存储）

```
MongoDB (mediarch_chunks)
├─ chunk_id (主键)
├─ content (文本内容)
├─ page_range (页码)
├─ position (坐标)
├─ source_path (PDF路径)
├─ doc_id (文档ID)
└─ entity_ids (关联的实体ID列表) ← 新增字段

Milvus (mediarch_chunks_vectors)
├─ chunk_id (关联 MongoDB)
└─ vector (向量嵌入)

Neo4j (知识图谱)
├─ 实体节点
│   ├─ id (实体ID)
│   ├─ name (实体名称)
│   ├─ type (实体类型)
│   ├─ chunk_ids (关联的 chunk_id 列表) ← 新增属性
│   └─ properties (关键属性)
└─ 关系边
    ├─ type (关系类型)
    └─ chunk_ids (支撑该关系的 chunk_id 列表) ← 新增属性
```

## 3. 实现方案

### 3.1 数据关联（构建双向链接）

#### 在图谱构建时，建立 chunk ↔ entity 关联

```python
# backend/databases/graph/builders/kg_builder.py

def _create_entity_with_chunks(self, entity_data: Dict, chunk_id: str):
    """创建实体节点，并关联 chunk_id"""

    # 创建实体节点
    entity_id = entity_data["id"]
    entity_name = entity_data["name"]
    entity_type = entity_data["type"]

    # 在 Neo4j 中存储 chunk_ids
    query = """
    MERGE (e:Entity {id: $entity_id})
    SET e.name = $entity_name,
        e.type = $entity_type,
        e.chunk_ids = CASE
            WHEN e.chunk_ids IS NULL THEN [$chunk_id]
            ELSE e.chunk_ids + $chunk_id
        END
    """

    self.neo4j_session.run(query,
        entity_id=entity_id,
        entity_name=entity_name,
        entity_type=entity_type,
        chunk_id=chunk_id
    )

    # 同时在 MongoDB chunk 中存储 entity_ids
    self.mongodb_chunks.update_one(
        {"chunk_id": chunk_id},
        {"$addToSet": {"entity_ids": entity_id}}
    )
```

### 3.2 混合检索器

```python
# backend/app/services/hybrid_retriever.py

class HybridRetriever:
    """混合检索器：向量 + 图谱"""

    def __init__(self):
        self.milvus_client = MilvusClient()
        self.neo4j_driver = Neo4jDriver()
        self.mongodb_client = MongoDBClient()

    def retrieve(self, query: str, k: int = 10) -> Dict[str, Any]:
        """
        混合检索

        Returns:
            {
                "chunks": [...],  # 检索到的文本块
                "entities": [...],  # 相关实体
                "relations": [...],  # 相关关系
                "graph": {...}  # 子图结构
            }
        """
        # 1. 并行检索
        vector_results = self._vector_search(query, k)
        graph_results = self._graph_search(query, k)

        # 2. 融合结果
        merged_chunks = self._merge_results(vector_results, graph_results)

        # 3. 构建知识网络
        knowledge_network = self._build_knowledge_network(merged_chunks)

        return knowledge_network

    def _vector_search(self, query: str, k: int) -> List[Dict]:
        """向量检索"""
        # 从 Milvus 检索相似 chunks
        vector_results = self.milvus_client.search(query, k)

        # 获取 chunks 中关联的 entity_ids
        chunk_ids = [r["chunk_id"] for r in vector_results]
        chunks = self.mongodb_client.find_chunks(chunk_ids)

        # 提取 entity_ids
        entity_ids = set()
        for chunk in chunks:
            entity_ids.update(chunk.get("entity_ids", []))

        return {
            "chunks": chunks,
            "entity_ids": list(entity_ids),
            "source": "vector"
        }

    def _graph_search(self, query: str, k: int) -> List[Dict]:
        """图谱检索"""
        # 1. 从查询中提取实体（使用 NER 或 LLM）
        entities = self._extract_entities(query)

        # 2. 在 Neo4j 中查找相关实体和关系
        cypher = """
        MATCH (e:Entity)
        WHERE e.name IN $entity_names
        OPTIONAL MATCH (e)-[r]-(related:Entity)
        RETURN e, collect(r) as relations, collect(related) as related_entities
        """

        results = self.neo4j_driver.run(cypher, entity_names=entities)

        # 3. 获取实体关联的 chunk_ids
        chunk_ids = set()
        for record in results:
            entity = record["e"]
            chunk_ids.update(entity.get("chunk_ids", []))

        # 4. 从 MongoDB 获取 chunks
        chunks = self.mongodb_client.find_chunks(list(chunk_ids))

        return {
            "chunks": chunks,
            "entities": [r["e"] for r in results],
            "relations": [r["relations"] for r in results],
            "source": "graph"
        }

    def _merge_results(self, vector_results: Dict, graph_results: Dict) -> List[Dict]:
        """融合向量和图谱检索结果"""
        # 1. 合并 chunks（去重）
        all_chunks = {}

        # 向量检索的 chunks（高相似度）
        for chunk in vector_results["chunks"]:
            chunk_id = chunk["chunk_id"]
            all_chunks[chunk_id] = {
                **chunk,
                "score_vector": chunk.get("similarity", 0),
                "score_graph": 0,
                "sources": ["vector"]
            }

        # 图谱检索的 chunks（高相关性）
        for chunk in graph_results["chunks"]:
            chunk_id = chunk["chunk_id"]
            if chunk_id in all_chunks:
                # 已存在，增加图谱分数
                all_chunks[chunk_id]["score_graph"] = 1.0
                all_chunks[chunk_id]["sources"].append("graph")
            else:
                # 新增
                all_chunks[chunk_id] = {
                    **chunk,
                    "score_vector": 0,
                    "score_graph": 1.0,
                    "sources": ["graph"]
                }

        # 2. 计算综合分数（向量 + 图谱）
        for chunk in all_chunks.values():
            chunk["score_hybrid"] = (
                0.6 * chunk["score_vector"] +  # 向量权重 60%
                0.4 * chunk["score_graph"]      # 图谱权重 40%
            )

        # 3. 按综合分数排序
        sorted_chunks = sorted(
            all_chunks.values(),
            key=lambda x: x["score_hybrid"],
            reverse=True
        )

        return sorted_chunks

    def _build_knowledge_network(self, chunks: List[Dict]) -> Dict[str, Any]:
        """构建知识网络（用于前端可视化）"""
        # 1. 提取所有实体
        entity_ids = set()
        for chunk in chunks:
            entity_ids.update(chunk.get("entity_ids", []))

        # 2. 从 Neo4j 获取实体和关系（子图）
        cypher = """
        MATCH (e:Entity)
        WHERE e.id IN $entity_ids
        OPTIONAL MATCH (e)-[r]-(related:Entity)
        WHERE related.id IN $entity_ids
        RETURN e, collect(DISTINCT r) as relations, collect(DISTINCT related) as related_entities
        """

        graph_data = self.neo4j_driver.run(cypher, entity_ids=list(entity_ids))

        # 3. 构建返回结构
        return {
            "chunks": chunks,
            "entities": [r["e"] for r in graph_data],
            "relations": [r["relations"] for r in graph_data],
            "graph": {
                "nodes": [{"id": e["id"], "name": e["name"], "type": e["type"]}
                          for r in graph_data for e in [r["e"]] + r["related_entities"]],
                "edges": [{"source": r["start"], "target": r["end"], "type": r["type"]}
                          for r in graph_data for rel in r["relations"] for r in [rel]]
            }
        }
```

## 5. 优势总结

### ✅ 互补增强
- **向量检索**：语义相似，召回率高
- **图谱检索**：结构化关系，精准度高
- **融合结果**：既有语义相关性，又有逻辑关联性

### ✅ 知识网络
- 用户不仅看到答案，还能看到**知识的上下文**
- 可视化图谱帮助理解**实体之间的关系**
- 推荐相关实体，促进**知识探索**

### ✅ 可追溯性
- 每个答案都有**原文出处**（PDF + 页码 + 坐标）
- 每个实体都有**支撑证据**（chunk_ids）
- 用户可以**验证答案的准确性**

## 7. 性能优化

### 缓存策略
- 缓存常见查询的图谱子图
- 缓存实体的 chunk_ids 映射

### 并行优化
- 向量检索和图谱检索完全并行
- 使用异步 I/O 减少等待时间

### 索引优化
- MongoDB: 为 `entity_ids` 字段建立索引
- Neo4j: 为 `chunk_ids` 属性建立索引
- Milvus: 使用 IVF_FLAT 索引加速检索

---

**总结**：这个混合架构将向量检索的语义能力和知识图谱的结构化能力完美结合，为用户提供更完整、更准确、更可追溯的知识服务。
